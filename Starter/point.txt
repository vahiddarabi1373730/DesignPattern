Facade:
ساختاری است که یک رابط ساده و یکدست روی یک زیرسیستم پیچیده می‌گذارد تا کلاینت لازم نباشد درگیر جزئیات و ترتیب صدا زدن چند کلاس مختلف شود.


یعنی:
پشت صحنه چند سرویس، چند مرحله، چند dependency و چند call داریم،
ولی به کاربر/کلاینت یک API ساده می‌دهیم.

برای مصاحبه:
 ساختاری است که یک interface ساده برای کار با یک subsystem پیچیده فراهم می‌کند.
 
یا:
 پیچیدگی چند کلاس یا چند سرویس داخلی را پشت یک API ساده پنهان می‌کند تا استفاده از سیستم راحت‌تر و coupling کمتر شود.
 
 
 فرض کن برای ثبت سفارش باید این‌ها را صدا بزنی:
 
 بررسی موجودی انبار
 محاسبه قیمت
 اعمال تخفیف
 ثبت پرداخت
 ساخت سفارش
 ارسال پیامک/ایمیل تایید
 اگر کلاینت خودش همه این‌ها را بداند، چند مشکل داریم:
 
 کلاینت به تعداد زیادی کلاس وابسته می‌شود
 ترتیب فراخوانی‌ها ممکن است اشتباه شود
 منطق orchestration در چند جا تکرار می‌شود
 استفاده از سیستم سخت می‌شود
 Facade می‌گوید:
 همه این شلوغی‌ها را بده به من،
 تو فقط یک متد مثل PlaceOrderAsync(...) صدا بزن.
